Using Position Values in CSS

How the CSS position property saved my relationship with my computer

November 27, 2014

Learning CSS has been challenging. There were moments of triumph when the CSS element that I added did exactly what I wanted it to do, but more often were the moments of utter confusion that left me throwing my arms in the air and yelling obscenities at my computer. (I’d like to take this moment to recommend learning CSS from the comfort and privacy of your home…you get a lot of odd looks when you’re cursing at your computer in Starbucks. Trust me.)

One of the battles I encountered when adding CSS to this website was matching the beautiful wireframes I’d designed. This became less of a struggle once I better understood the different position property values.

position:static

This is the default position value of all elements. There is actually no positioning involved with this value, so you could also think of it like it’s position:none. Static is best used for single column layouts where each element sits below the previous element. This is the only position value you’re unable to reposition using top, right, bottom, and left offset properties.

position:relative

The relative value behaves much like the static value until you add a top, right, bottom, or left offset property. Adding any of these will adjust the position of the section you’re working with, using its normal (original) placement as the starting point.

position:absolute

You can place an element absolutely anywhere using the absolute value. It does not follow the normal flow of a document. You can use the top, right, bottom, and left offset properties to define its dimensions without the use of height and width properties, which adds the bonus of stretching/shrinking as you change the size of your screen.

position:fixed

The fixed value allows you to keep an element in the same position on the screen when scrolling up/down and left/right. This is best for any information that is critical for the user to see and be able to access at any time. I’ve noticed that some sites will use this for their header/footer, advertisements, and in some cases links to their social media accounts.
