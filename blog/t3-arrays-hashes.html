Let’s Hash It Out!

Arrays vs. hashes

December 4, 2014

This week we took a look at using arrays and hashes. These are both techniques for creating lists in Ruby, but they are used in different circumstances.

Arrays

An array is an ordered list of objects. Each object is linked to its position within the list in numerical order (starting with ‘0’). You can apply a method (action) to an array to do things like sort the list (.sort) or reverse the order (.reverse), and you can also access specific objects from your list by referencing their positions in the list. Arrays are great if you’re planning on manipulating the objects in the list in some way.

Let’s say you wanted to work with a list the characters in the movie “Frozen”. frozen = [‘Elsa’, ‘Anna’, ‘Olaf’, ‘Kristoff’, ‘Hans’, ‘Sven’] You could apply methods to count the number of characters listed (frozen.count), alphabetize the characters (frozen.sort), find the first character listed (frozen.first), and so on. (Click here to view additional array methods.)

Hashes

A hash is a list of key and value pairs, with both the key and the value being objects. Just as an object in an array can be accessed using its numerical position, a value can be accessed within a hash using its key. Hashes are sometimes likened to dictionaries, since the values (definitions) can be found if you know the correct key (dictionary entry) to look up. Hashes allow you to store more information than an array would, and even allows you to list an array as a value.

Let’s create a hash for “Frozen” that includes more than just the list of characters. frozen = { characters => [‘Elsa’, ‘Anna’, ‘Olaf’, ‘Kristoff’, ‘Hans’, ‘Sven’] genre => “animation” released => 2013 runtime => “102 min” rating => “PG” setting => “Arendelle” songs => [“Let It Go”, “Do You Want to Build a Snowman?”, “For the First Time in Forever”, “Fixer Upper”] } Now you can lookup the movie’s rating (frozen[rating]) and figure out if a song is included as a value (frozen.has_value?
